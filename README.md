# 儿童成长冒险日历 Skill

把暑假、寒假或日常课程表，整理成一套适合孩子使用的“成长冒险日历”：支持逐项访谈、日期排期、每日趣味文案、计时与补签、经验值与兑换币、角色升级、家庭多设备同步，以及上线前测试。

这不是一个固定主题的网站模板，而是一套可复用的产品与工程方法。视觉风格由每个家庭自己定义；课程、奖励和角色数据也都放在配置层，方便更换。

## 能做什么

- 先读取家长提供的聊天记录、课表和日期限制，再一次只追问一个关键问题。
- 先产出“重复规则表 + 完整逐日计划”，确认后才开始写网页。
- 自动检查日期遗漏、课程冲突、任务编号、奖励、兑换物与角色等级。
- 生成适配手机、平板和电脑的日视图/周视图。
- 按真实时间锁定任务：未来不能提前完成，固定课程结束后才能领取，计时任务必须达到时长。
- 支持补签、经验值升级、兑换币商城、角色成长，以及可选的家庭云同步。
- 把日程数据、主题、游戏状态和同步服务分开，后续换课或换风格时不必推倒重做。

## 兼容环境

同一份 Skill 可用于：

- **Codex**：项目级 Skill 放在 `.agents/skills/`。
- **Qwen Code**：项目级 Skill 放在 `.qwen/skills/`。

Skill 本体在本仓库的 `kid-adventure-planner/` 目录中。

## 安装

### Codex

在你的项目根目录执行：

```bash
mkdir -p .agents/skills
cp -R /path/to/this-repo/kid-adventure-planner .agents/skills/
```

也可以直接下载仓库后，手动把 `kid-adventure-planner` 文件夹复制到项目的 `.agents/skills/`。

### Qwen Code

在你的项目根目录执行：

```bash
mkdir -p .qwen/skills
cp -R /path/to/this-repo/kid-adventure-planner .qwen/skills/
```

也可以把它安装为个人 Skill：将整个文件夹复制到 `~/.qwen/skills/`。

> 如果你的工具版本尚不支持 Skill，可以先打开 `kid-adventure-planner/SKILL.md`，把其中的流程作为项目提示词使用；参考资料仍可按文件名继续提供给模型。

## 使用

### Codex 示例

```text
$kid-adventure-planner

请帮我做一份 7 月 1 日到 8 月 31 日的暑假成长日历。
孩子 9 岁，上午学习、下午运动，需要手机和平板同步。
先看我提供的课表和聊天记录，一次只问我一个关键问题，
在我确认完整逐日计划之前不要写网页。
视觉主题先不要替我决定。
```

### Qwen Code 示例

```text
/kid-adventure-planner

请按这个 Skill 帮我做一份儿童寒假冒险日历。
先完成需求访谈和逐日排期，再做本地版本；
等我确认后再增加计时、奖励和家庭同步。
```

建议按三档逐步做：

1. **计划版**：冲突检查后的完整日程。
2. **单设备版**：响应式页面、计时、奖励、本地保存。
3. **家庭同步版**：服务器校验、持久化数据库、手机和平板同步。

## 配置校验

需要本机安装 Node.js。完成 `planner-config.json` 后，在仓库根目录运行：

```bash
node kid-adventure-planner/scripts/validate-plan.mjs path/to/planner-config.json
```

校验器会检查：

- 假期日期是否完整且无重复；
- 任务 ID 是否稳定且唯一；
- 固定课程和计时任务是否有合法时长；
- 家庭日、旅行日、休息日和纯信息任务是否错误发放奖励；
- 趣味文案是否缺失或重复；
- 每周可获得的兑换币是否足以兑换当周物品；
- 商店、角色和等级阶段是否完整。

所有 `ERROR` 必须修复；`WARNING` 需要由家长确认后再继续。

## 架构

项目建议保持四层独立：

1. `planner-config`：日期、任务、趣味文案、奖励、商城、角色阶段。
2. `theme-config`：颜色、字体、卡片、图标、动效和声音映射。
3. `game-state`：完成记录、计时、购买、角色选择、数据版本。
4. `sync-service`：家庭身份、服务器校验、持久化存储、合并结果。

这能确保“调整课表”“重做皮肤”“修改奖励”和“修复同步”分别发生在正确的层，不会互相污染。

## 必须保留的产品规则

- 未来任务锁定；固定课程到结束时间后才可完成。
- 计时任务以可信的开始时间和配置时长判断，不相信客户端自报时长。
- 过去的合格任务可以补签，但必须明确标记。
- 家庭日、旅行日、休息日和纯信息条目不产生奖励。
- 经验值只用于升级；兑换币只用于购买。
- 领取和购买都必须幂等，同一个任务或物品不能重复获利。
- 有云同步时，服务器重新计算时间、奖励、价格和解锁条件。
- 领取成功前保留计时证据；同步过程中产生的新操作不能被旧响应覆盖。
- 多设备按记录合并，不得整包覆盖状态。
- 已上线任务的 ID 不随改课而重建，否则会丢失历史。

完整规则见：

- `kid-adventure-planner/references/product-rules.md`
- `kid-adventure-planner/references/implementation.md`
- `kid-adventure-planner/references/testing.md`

## 可以自定义什么

以下内容适合按家庭喜好自由定义：

- 主题名称、世界观、配色、字体、卡片和按钮；
- 原创角色、等级外观、兑换物、图标、动效和音效；
- 课程、作息、趣味文案、奖励数值和每周商城；
- 家长确认方式、补签规则和是否需要云同步；
- 部署平台与数据库。

以下逻辑不建议为了视觉效果而删改：时间锁、计时证据、幂等领取、服务器权威、记录级合并、数据迁移和回归测试。

## 隐私与版权

- 公开演示只使用虚构昵称和虚构日程，不写学校、班级、住址、老师联系方式或精确场馆。
- 家庭同步码应当视为私钥；服务端存储时尽量散列处理。
- 私人日历应设置 `noindex`，并严格隔离不同家庭的数据。
- 本 Skill 不包含第三方游戏贴图、角色、Logo、音乐或音效。请只使用原创、已授权、公有领域或用户自己提供的素材。
- “仅供学习”或版权声明并不会自动获得素材再分发权。
